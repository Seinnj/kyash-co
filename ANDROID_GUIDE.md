# 📱 Androidでの使用ガイド

## 🚀 クイックセットアップ

### ステップ1️⃣: PC側でサーバーを起動

```bash
npm install
npm start
```

**表示されるメッセージ（重要）:**
```
🔗 アクセスURL:
   • PCから:        http://localhost:3000
   • スマートフォンから: http://192.168.x.x:3000
📱 Androidの設定方法:
   1. PC側のIPアドレス: 192.168.x.x
   2. ポート番号: 3000
```

**IPアドレスをメモしてください！**（例: `192.168.1.100`）

---

### ステップ2️⃣: Androidデバイスの設定

#### 準備
- AndroidデバイスをPCと同じWi-Fiネットワークに接続
- PCのIPアドレスを確認（上記のメッセージから）

#### ブラウザでアクセス

Android Chrome/Firefoxなどのブラウザを開いて以下を入力：

```
http://192.168.1.100:3000/login.html
```

※ `192.168.1.100` はPC側のIPアドレスに置き換えてください

---

## ⚙️ サーバーURL設定

ログイン画面の上部に**「サーバーURL設定」**が表示されます。

```
📍 サーバーURL設定
[入力フィールド] [設定ボタン]
例: http://192.168.1.100:3000
```

### 設定方法：

1. **PC側のIPアドレスを入力**
   - 例: `http://192.168.1.100:3000`

2. **「設定」ボタンをクリック**

3. **完了**
   - 以降は自動的にそのサーバーを使用します
   - 再度変更する場合は同じフォームで新しいIPを入力

---

## 🔍 IPアドレスの確認方法

### PC側でIPアドレスを確認

#### **Windowsの場合:**
```bash
ipconfig
```
または
```bash
コマンドプロンプトで：ipconfig /all
```

「IPv4 アドレス」を確認（例：192.168.1.100）

#### **Macの場合:**
```bash
ifconfig
```

「inet」で始まる行を確認

#### **Linuxの場合:**
```bash
ip addr show
```

または

```bash
hostname -I
```

#### **Node.jsサーバーの出力から確認：**
```
🔗 アクセスURL:
   • スマートフォンから: http://192.168.1.100:3000
```

## 🐛 トラブルシューティング

### ❌ ページが開けない

**原因1: IPアドレスが間違っている**
- PC側で `npm start` を実行時に表示されるIPアドレスを確認
- Androidで同じIPアドレスを入力

**原因2: Wi-Fiが接続されていない**
- AndroidをPCと同じWi-Fiネットワークに接続

**原因3: ファイアウォールがブロック**
- PC側のファイアウォール設定を確認
- ポート3000を許可する

### ❌ サーバーに接続できない

**確認1: PC側でサーバーが起動しているか**
```bash
npm start
```
実行して、メッセージが表示されているか確認

**確認2: PC側のIPアドレスを再度確認**
```bash
# Windowsの場合
ipconfig
```

**確認3: Androidの設定ページでURLを入力し直す**
ログイン画面の「サーバーURL設定」で再度設定

### ❌ Telegramに通知が届かない

**確認: PC側でサーバーが正常に動作しているか**
- ターミナルにエラーメッセージがないか確認
- `http://{IPアドレス}:3000/health` にアクセスして確認

## 💡 Tips

### 複数デバイスからのアクセス

同じPC側のサーバーに複数のAndroidデバイスからアクセス可能です：
- スマートフォン1: `http://192.168.1.100:3000/login.html`
- スマートフォン2: `http://192.168.1.100:3000/admin.html`
- タブレット: `http://192.168.1.100:3000/login.html`

### ローカルストレージについて

一度設定したサーバーURLはAndroid側に保存されます：
- 同じブラウザで再度アクセスすると自動的に接続
- キャッシュをクリアすると設定が削除されます

### 動的ホスト名の場合

PC側が頻繁にIPアドレスを変える場合：
- ルーターでPCに固定IPアドレスを割り当てる
- または、ホスト名でアクセス（ネットワーク設定による）

---

## 🔐 セキュリティ注意

⚠️ **重要:**
- 同じWi-Fi上のすべてのデバイスからアクセス可能
- ローカルネットワーク内のみで使用
- 本番環境では適切なセキュリティ対策を実装してください

---

## 📞 サポート

問題が解決しない場合：
1. PC側のサーバーのエラーメッセージを確認
2. IPアドレスが正しいか再確認
3. Wi-Fi接続を確認
4. ブラウザのキャッシュをクリア
